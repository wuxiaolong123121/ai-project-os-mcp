# AI Project OS MCP 零技术快速入门

## 一、什么是 AI Project OS？

AI Project OS 是一个**工程级 AI 行为治理引擎**，它能让 AI 像一支专业的工程团队一样工作，而不是一个随心所欲的聊天机器人。

## 二、为什么需要 AI Project OS？

当你使用 AI 写代码时，是不是遇到过这些问题：

- AI 跳过规划，直接写代码
- AI 随意修改架构
- AI 打补丁而不是重新设计
- 你不知道 AI 什么时候越界了

AI Project OS 解决了这些问题，它让 AI 必须遵循**真实的软件工程流程**。

## 三、零技术使用指南

### 3.1 准备工作

你只需要：

1. 一个支持 MCP 的 AI 工具（如 Claude、Cursor、Trae）
2. 一个 GitHub 账号（用于获取项目代码）
3. 对软件工程有基本概念

### 3.2 从想法到可交付项目的完整流程

#### 步骤 1：理解 5S 工作流

AI Project OS 使用 **5S 工作流**来管理项目，每一步都有明确的规则：

| 阶段 | 含义 | 代码允许 | 主要任务 |
|-----|-----|---------|---------|
| S1 | Scope（范围） | ❌ | 定义项目边界 |
| S2 | Spec（规格） | ❌ | 编写详细需求 |
| S3 | Structure（结构） | ❌ | 设计架构 |
| S4 | Schedule（计划） | ❌ | 制定开发计划 |
| S5 | Ship（交付） | ✅ | 生成代码并测试 |

**核心规则**：只有在 S5 阶段，AI 才能生成代码！

#### 步骤 2：启动第一个项目

1. **打开你的 AI 工具**（如 Claude）
2. **告诉 AI**："我们使用 AI Project OS MCP 来开发项目，请遵循 5S 工作流"
3. **开始 S1 阶段**：
   - 告诉 AI 你的项目想法
   - AI 会帮你定义清晰的项目范围
   - 你确认后，范围被"冻结"

#### 步骤 3：完成 S2-S4 阶段

1. **S2（规格）**：AI 帮你编写详细的功能需求
2. **S3（结构）**：AI 设计项目架构和文件结构
3. **S4（计划）**：AI 制定开发计划和任务分解

**重要**：每完成一个阶段，AI 都会要求你确认，确认后该阶段就被"冻结"，不能再随意修改。

#### 步骤 4：进入 S5 阶段，生成代码

当 S4 阶段完成并冻结后，AI 会自动进入 S5 阶段，开始生成代码。

在 S5 阶段，AI 会：

1. 按照计划生成代码
2. 自动运行测试
3. 生成完整的审计记录
4. 确保代码符合架构设计

#### 步骤 5：获取最终交付物

完成 S5 阶段后，你会得到：

- 可运行的代码
- 完整的项目文档
- 详细的审计记录
- 测试报告

### 3.3 治理效果展示

| 传统 AI 开发 | AI Project OS 治理开发 |
|-------------|----------------------|
| AI 跳过规划直接写代码 | 必须完成 S1-S4 才能进入 S5 |
| AI 随意修改架构 | 架构冻结后不能修改，只能重新设计 |
| 无审计记录 | 每一步都有详细的审计日志 |
| 质量不可控 | 自动运行测试，确保质量 |

## 四、常见问题

### 4.1 我需要写代码吗？

不需要！你只需要：
- 提出你的想法
- 确认每阶段的成果
- 验收最终交付物

### 4.2 我需要安装软件吗？

不需要！AI Project OS 作为 MCP 服务运行在云端，你只需要通过支持 MCP 的 AI 工具访问它。

### 4.3 我能控制 AI 吗？

是的！你通过**冻结机制**控制 AI：
- 只有你确认的内容才能进入下一阶段
- 你可以拒绝不满意的结果
- 你始终掌握项目的最终决定权

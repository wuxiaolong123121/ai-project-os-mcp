# AI Project OS MCP 快速开始（完全不懂技术也能懂）

## 你是谁？

如果你：
- 完全不懂代码
- 用过 ChatGPT / 豆包 / Claude
- 想让 AI 帮你做项目，但害怕失控

那这篇指南就是为你准备的！

## 一句话核心

**你不需要懂代码，只需要会做判断。**

AI Project OS MCP 会管住 AI，不让它乱来。

## 第一步：初始化项目

1. **下载项目**
   - 点击 GitHub 仓库右上角的「Code」按钮
   - 选择「Download ZIP」
   - 解压到你喜欢的文件夹

2. **安装 Python**
   - 从 Python 官网下载并安装 Python 3.10 或以上版本
   - 安装时记得勾选「Add Python to PATH」

3. **启动 AI Project OS**
   - 打开文件夹，在地址栏输入「cmd」，按回车
   - 在命令窗口中输入：`python setup_ai_os.py`
   - 按回车，等待执行完成

## 第二步：了解项目结构

初始化完成后，你会看到一些文件夹：

| 文件夹/文件 | 作用 | 你需要管吗？ |
|------------|------|-------------|
| `docs/` | 项目文档 | 可以看看，了解更多 |
| `examples/` | 示例文件 | 建议先看 `example-no-code.md` |
| `prompts/` | AI 提示词 | 核心文件，不要乱改 |
| `scripts/` | 自动化脚本 | 系统用的，不要乱改 |
| `state.json` | 项目状态 | 系统自动管理，不要乱改 |
| `ai_project_os_mcp/` | MCP SDK | 系统核心，不要乱改 |

## 第三步：开始使用 MCP

### 1. 让 AI 成为「守规矩的执行者」

- 打开 `prompts/mother_prompt.md`
- 复制所有内容
- 粘贴到你喜欢的 AI 工具（ChatGPT / 豆包 / Claude）
- 对 AI 说：「我们现在开始 S1，请你等我给 Scope。」

### 2. 定义你的项目（S1）

对 AI 说：
```
我想要一个 [你的项目想法]，
不做 [你明确不想要的功能]，
成功的标准是 [你觉得什么时候算成功]。
```

例如：
```
我想要一个自动生成短视频脚本的工具，
不做视频剪辑，不接平台 API，
成功的标准是输入主题能输出完整的短视频脚本。
```

### 3. 冻结 S1

AI 会帮你整理成清晰的 Scope 文档。

你只需要判断：**这是不是我想要的？**

- 如果是：对 AI 说「冻结 S1」，然后运行 `python scripts/freeze_stage.py S1`
- 如果不是：让 AI 重写，直到你满意

### 4. 重复这个流程

接下来的 S2-S5 阶段，你只需要：

1. 让 AI 生成对应阶段的内容
2. 判断：**这是不是我想要的？**
3. 冻结阶段（运行 `python scripts/freeze_stage.py S2` 等）
4. 进入下一阶段

## 第四步：让 MCP 管住 AI

MCP 会自动做这些事：

- ❌ 禁止 AI 提前写代码（只有 S5 才能写）
- ❌ 禁止 AI 跳阶段
- ❌ 禁止 AI 绕过架构
- ✅ 强制 AI 写审计记录

你不需要管这些，MCP 会自动处理！

## 第五步：AI 写代码（S5）

当你冻结到 S5 阶段后：

1. AI 会自动开始写代码
2. 每写一个功能，AI 都会：
   - 说明在干什么
   - 说明什么是「对的」
   - 留下审计记录
3. 你只需要看：**它有没有越权？**

## 常见问题

### Q: 我需要安装其他软件吗？
A: 只需要 Python，其他都不需要。

### Q: 我可以用中文和 AI 交流吗？
A: 当然可以！AI 完全支持中文。

### Q: 项目失败了怎么办？
A: 没关系！
   - 你可以清楚地知道失败在哪一步
   - 可以重新开始，不会浪费时间
   - 你永远不会被 AI 拖着跑

## 小技巧

1. **先从简单项目开始**：比如生成文章、整理数据等
2. **不要贪心**：一次只做一个功能
3. **多冻结**：每个阶段都要冻结，不要跳过
4. **看示例**：先看看 `examples/example-no-code.md`，跟着做一遍
